# 澄心守望——基于多模态大模型的高校心理陪伴与危机预警平台

## 项目简介
澄心守望面向高校心理健康教育场景，构建学生端、心理辅导工作台与系统治理中心协同运行的一体化平台。系统围绕日常陪伴、情绪记录、风险预警、个案跟进和平台治理五类核心任务展开，支持学生心理支持服务的连续化、规范化与留痕化管理。

## 核心功能
### 学生端
- 心理对话：支持日常倾诉、主题引导、风险识别与即时帮助提示。
- 情绪日记：支持文本记录、情绪维度分析、关键词提取与阶段趋势展示。
- 呼吸训练：提供稳定情绪的引导式放松模块。

### 心理辅导工作台
- 预警总览：展示高风险、中风险、待研判、跟进中等关键指标。
- 风险处置：支持按状态查看、更新预警，形成分级处置链路。
- 个案管理：查看学生档案、对话记录、日记记录、预警记录与干预记录。
- 群体画像：统计近阶段高频压力关键词与风险来源分布。

### 系统治理中心
- 平台总览：查看学生、对话、日记、预警、干预等总体数据。
- 数据治理：支持数据导入、导出、重置与脱敏管理。
- 运行维护：展示核心服务运行状态与业务趋势。

## 技术架构
- 前端：React + Vite + TypeScript
- 后端：Express + TypeScript
- 数据存储：JSON 文件化存储
- 分析服务：对话回复、日记分析、风险识别统一由后端服务层处理

## 启动方式
### 前端
```bash
cd frontend
npm install
npm run dev
```

### 后端
```bash
cd backend
npm install
cp .env.example .env
# 按实际部署环境填写访问口令与服务配置
npm run dev
```

## 环境变量说明
后端运行前需在 `.env` 中配置：
- `PORT`：后端服务端口
- `CORS_ORIGIN`：前端访问地址
- `ADMIN_PASSWORD`：心理辅导工作台访问口令
- `SUPERADMIN_PASSWORD`：系统治理中心访问口令
- `AUTH_SECRET`：签名密钥
- `DATA_FILE`：数据文件路径
